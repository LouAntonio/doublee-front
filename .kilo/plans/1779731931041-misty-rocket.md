# Plano: Refatorar Página de Gestão (/dbe) com Consistência do /sobre

## Contexto Atual

A página `/dbe` (painel administrativo) usa um design genérico Tailwind slate — sem personalidade de marca. A página `/sobre` tem uma identidade visual forte: fundo sand quente, Fredoka para títulos, Nunito para corpo, laranja como cor de destaque, e patterns orgânicos.

## Direção Estética

**"Warm Editorial Admin"** — Um painel administrativo premium que herda a alma visual da página Sobre, adaptada para contexto denso em dados. Como o back-office de uma revista de design.

## O que será alterado

### 1. AdminDashboardLayout.jsx (LAYOUT PRINCIPAL)
- **Background**: `bg-slate-50` → `bg-sand` + `geo-pattern`
- **Sidebar**: manter dark (#0f172a) mas usar `font-display` no logo
- **Header**: usar fundo `white/80 backdrop-blur` com borda `sand` em vez de `slate-200`
- **Page title**: usar `font-display` + cor `text-[#1C1917]`
- **Decorativos**: mudar gradientes azuis/indigo para laranja (accent)
- **Scrollbar**: manter mas com laranja suave

### 2. AdminDashboard.jsx (DASHBOARD PRINCIPAL)
- **Título "Visão Geral"**: `font-display` + `text-[#1C1917]`
- **Stat Cards**: fundo `bg-white` com borda `sand`, hover com sombra laranja
- **Ícones**: substituir gradientes azuis/roxos por tons alinhados ao brand
- **Tabelas**: cabeçalhos com fundo `bg-[#F5F0EB]/80` em vez de `bg-slate-50`
- **Links hover**: `text-orange-600` em vez de variações de indigo/blue
- **Remover inline styles duplicados** (fadeInUp já existe no index.css como animate-fade-in-up)

### 3. AdminUsers.jsx
- **Page title**: `font-display` + cor consistente
- **Status badges**: usar tons laranja/amber em vez de azul/indigo
- **Botões de ação**: padronizar com laranja/accent
- **Header card**: fundo sand em vez de branco puro
- **Tabela**: borders sand, cabeçalho sand
- **Skeleton**: usar sand em vez de slate
- **Empty states**: ícones consistentes

### 4. AdminStores.jsx
- **Títulos**: `font-display`
- **Modal de detalhes**: fundo sand sections em vez de `bg-slate-50`
- **Botões de ação**: brand orange
- **Tabela**: padrão sand
- **Status indicators**: manter lógica mas usar tons consistentes

### 5. AdminProducts.jsx
- **Filters area**: sand background, brand typography
- **Status modal**: sand sections
- **Details modal**: sand sections
- **Tabela**: padrão sand

### 6. AdminCategories.jsx
- **Form/Header**: sand bg
- **Edit modal**: sand bg, accent focus rings
- **Botões**: brand orange

### 7. AdminPromotions.jsx
- **Form card**: sand bg
- **Tabela**: sand header

### 8. AdminIdentityVerification.jsx
- **Detail modal**: sand sections
- **Tabela**: sand header
- **Filter pills**: sand bg, active com brand orange

### 9. AdminAnalytics.jsx
- **Stat cards**: sand palette
- **Atividade**: sand borders

## Principais Mudanças Visuais (Resumo)

| Elemento | Antes | Depois |
|---|---|---|
| Background | `bg-slate-50` | `bg-sand` / `bg-[#F5F0EB]` |
| Títulos | `text-slate-800 font-bold` | `text-[#1C1917] font-display` |
| Texto muted | `text-slate-500` | `text-[#78716C]` |
| Borda cards | `border-slate-100` | `border-[#E8E0D8]` |
| Ícone card hover | gradiente azul/roxo | laranja/accent |
| Inline styles | `<style>` duplicado em 7 páginas | Removido, usando `animate-fade-in-up` do Tailwind via index.css |
| Foco inputs | `ring-slate-400` / `ring-orange-500` | `ring-accent/20` |

## Anexo: Esquema de Cores Consistente

```css
/* Do index.css */
--color-sand: #F5F0EB;
--color-accent: #F97316;
--color-accent-dark: #EA580C;
--font-display: "Fredoka", sans-serif;
--font-body: "Nunito", sans-serif;

/* Uso nas páginas admin */
.bg-page        → bg-sand
.bg-card        → bg-white
.bg-card-header → bg-[#F5F0EB]/80
.border-card    → border-[#E8E0D8]
.text-title     → text-[#1C1917] font-display
.text-body      → text-[#78716C]
.text-accent    → text-accent
```

## Não Alterar

- Lógica de negócio, queries, mutations, handlers
- Estrutura de componentes e props
- Comportamento de modais e paginação
- Navegação e rotas
- Funcionalidade existente
